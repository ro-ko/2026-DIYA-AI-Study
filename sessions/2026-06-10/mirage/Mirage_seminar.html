Problem

RGB point-cloud memory의 두 가지 본질적 병목

3D 일관성을 위해 기존 world model은 RGB 색상의 명시적 point cloud를 메모리로 유지 — 매 conditioning step마다 pixel space를 왕복한다.

Cache M_rgb
→ Rasterise
→ Shade (HW)
→ VAE Encode
→ ẑₜ (latent)
매 target view마다 강제되는 round trip:  ẑₜ = E( Rasterise( M_rgb ; Eₜ, Kₜ ) )
1

연산 비용 (Compute)

  • Rasterise는 pixel 해상도 HW에서 수행되지만 생성기는 latent hw를 소비
  • 수백만 colored point를 full-res 렌더 + VAE 재인코딩 → wall-clock 지배
  • cache가 커질수록 매 step 비용이 선형 증가
2

표현 손실 (Fidelity)

  • RGB로 렌더 후 다시 encode한 surrogate signal은 backbone native latent에서 이탈
  • VAE reconstruction error · rasterization artifact
  • visibility hole · distribution mismatch 누적

Method · Formulation

세 개의 핵심 연산

Eq.4 · Construction

p_uv = π⁻¹( u, v, D(u,v) ; K, E )    F_uv = z[ :, v, u ]
depth를 latent grid로 downsample, K rescale. 각 latent cell을 world로 back-projection하고 해당 latent token을 부착.

Eq.5 · Readout (z-buffer)

iₜ(u,v) = argmini∈Ωₜ [Eₜ pᵢ]_z    ẑₜ(u,v) = Fiₜ(u,v)
target grid로 투영, frontmost point만 retain. 빈 cell은 zero-fill + binary visibility mask mₜ. bridging encoder 불필요.

Eq.6 · Cache Update

M ← M ∪ { (p_uv, F_uv) }(u,v) ∈ Λₜ
Λₜ = valid depth & 동적객체/하늘 밖 cell (Qwen3-VL + SAM3). transient content가 persistent cache를 오염시키는 것 방지.

Analysis · Ablation

각 컴포넌트의 기여 (WorldScore split)

Latent > RGB

동일 조건 RGB로 교체 시 일관성 하락 → RGB detour가 정보 폐기

Geometry vs feature downsample

feature를 pixel 해상도로 lift하면 pretraining 분포 밖 interpolation → 3D 악화

Dynamic filter 필수

끄면 stale 동적 content가 future chunk로 splatting → long-horizon 불안정

Depth source robustness

DepthAnything3 70.36 → MapAnything 69.66 → UniDepth 69.13 (soft hint)

Conclusion

Latent space로 cache를 옮기는 것만으로
품질과 효율을 동시에 잡는다

Contribution — latent feature를 world 좌표에 저장하는 최초의 latent-space 3D memory
Mechanism — 단일 latent-resolution projection으로 readout → rasterise-encode 왕복 제거, cache footprint는 s² 배 축소
Limitation — dynamic filter로 움직이는 객체 제외 → 동적 actor 상태는 chunk 간 미유지. pervasive motion scene은 이득 적음
